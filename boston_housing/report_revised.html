Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Previously: Grid search is a brute force method to locate the optimal hyperparameters. Rather than other programming methods, grid search computes the results of the model using every combinations/possibilities for a set of hyperparameters, and select the values that yield the optimal outcome. This exhaustive method forms a grid in the hyperparameter space. The above figure is an example of grid search for an optimal max depth in a decision tree.

Revised Response:

Grid Search allows us to identify an optimal set of hyperparameters by providing a pool of various models. This exhaustive search method adjusts each hyperparameter and tests the model with all the possible combinations of hyperparameter values. The optimal set of hyperparameters is defined as the hyperparameters that yield the optimal outcome. Here the optimal outcome can be measured through metrics such as F-1 score or R-square, or some other metrics, or a combination of these scores, depending on the individual problems.

Grid search is convinient in a way that we do not have to conduct an in-depth analysis on the hyperparameter space w.r.t the model. This is different from other programming approaches such as a gradient search method to directly find the values that optimizes the cost/evaluation metrics. Grid search is advantageous when the model is highly nonlinear, contains noncontinuity or stochasticity, or there is no closed form derivation (or very hard to perform mathematical analysis). But grid search also suffers problems like heavier computational burden, unknown hyperparameter testing range (how do we ensure the testing range indeed supersets the optimal), unknown hyperparameter testing stepsize for continuous hyperparameters (what stepsize to choose: 1? 0.1? 0.0001?), etc.

As for a dicision tree classifier, grid search method is probably the best way to estimate the hyperparameters, since most of them are finite, discrete valued. An example of grid search in this case is to test the model by exhaustively trying all the possible combinations like min_samples_leaf=[2,4,8], min_samples_split=[2,4,6],max_depth:[1,2,3]}. Once we complete the search, we should have a set of optmal hyperparameters for this model.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(n_splits= 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor(random_state=0)

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {}
    params['max_depth'] = list(range(1,11))

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV():
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,param_grid = params,scoring = scoring_fnc,cv = cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Sensitivity¶
